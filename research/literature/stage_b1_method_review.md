# Stage B1 自由终端时间与网格收敛方法评审

## 1. 范围、证据等级与不可替代基线

本评审服务于 Stage B1：在保留固定终端时间凸子问题的前提下，研究终端时间搜索、离散网格收敛和物理可行性。它不是对现有 `400.7 kg` 基准的重命名，也不把探索结果提前写成论文结论。

`MarsLanding/MarsLanding.c` 是作者原始、独立、人工完成的 CRS→CCS 手写实现。在 Stage B1 中应将其登记为 `legacy_tf81_v1`：`N=30`、`t_f=81 s`，继续作为历史数值一致性和独立工程基线。新建的参数化、自动建模或自由终端时间模型只能与它并列交叉验证，不能覆盖、导出、生成或替换它，也不能反向声称原始手写实现来自自动模型。

本文使用两级证据：

- `仓库已有来源`：已经进入 `paper/refs.bib` 和系统文献矩阵，但矩阵目前多数仅完成元数据核验；本文只作与题名和书目信息相符的保守归类。
- `专项元数据核验`：2026-07-13 通过 Crossref DOI 记录核验题名、作者、年份和出版物。除非另有说明，不据此声称算法细节、定理条件或性能优势，正式引用前仍须全文逐页提取。

## 2. 文献地图

### 2.1 固定终端时间凸化是 Stage B1 的可信内核

Acikmese 与 Ploen 的 Mars 动力下降凸规划工作，以及后续软着陆控制界和指向约束的无损凸化工作，是本项目固定 `t_f` SOCP 路径的直接背景（DOI `10.2514/1.27553`、`10.1109/TCST.2012.2237346`，仓库已有来源）。Blackmore 等关于控制约束无损凸化的工作进一步属于该理论谱系（DOI `10.1016/j.sysconle.2012.04.010`，仓库已有来源）。

这些来源能支持“本项目属于凸动力下降制导谱系”这一定位，但不能单凭元数据支持以下更强结论：当前离散化严格满足某篇论文的全部无损条件；自由 `t_f` 后整体问题仍是单个 SOCP；或者离散解已经代表连续时间最优解。Stage B1 必须对这些命题分别验证。

### 2.2 自由终端时间已有直接相关工作，简单外层搜索不是空白领域

专项检索发现至少两篇题名直接指向自由终端时间动力着陆：

| 来源 | 元数据可确认的范围 | Stage B1 启示 |
|---|---|---|
| Yang 与 Liu, *Fuel-optimal powered descent guidance with free final-time and path constraints*, Acta Astronautica (2020), DOI `10.1016/j.actaastro.2020.03.025` | 题名、作者、年份、期刊已由 Crossref 核验 | 投稿前必须全文比较其终端时间处理、路径约束和求解结构；不能把“外层一维搜索”本身作为原创贡献 |
| Li 与 Gong, *Free Final-Time Fuel-Optimal Powered Landing Guidance Algorithm Combing Lossless Convex Optimization with Deep Neural Network Predictor*, Applied Sciences (2022), DOI `10.3390/app12073383` | 题名、作者、年份、期刊已由 Crossref 核验 | 学习型终端时间预测已是相邻路线；本项目若不采用学习器，应突出确定性证据、可审计搜索和嵌入式结构复用，而非宽泛声称首次自由时间制导 |

因此，Stage B1 的近期目标应是建立可信参考解和误差预算，而不是把外层搜索包装成核心原创算法。潜在贡献需要来自更完整的组合：固定结构参数更新、经证明或经充分实验限定的搜索策略、网格误差控制、跨求解器一致性，以及嵌入式重复求解证据。

### 2.3 网格收敛不能由三个燃耗数字代替

Sagliano 的 *Pseudospectral Convex Optimization for Powered Descent and Landing*（DOI `10.2514/1.G002818`）和 *Generalized hp Pseudospectral Convex Programming for Powered Descent and Landing*（DOI `10.2514/6.2018-1870`）在题名层面直接连接凸动力下降与伪谱/hp 离散化；Le Merrer 等的直接转录网格细化工作（DOI `10.3182/20110828-6-IT-1002.01577`）在题名层面直接涉及 mesh refinement。三者均已完成专项元数据核验，算法细节和误差准则仍需全文提取。

仓库已有 Hargraves--Paris 直接配点来源（DOI `10.2514/3.20223`）和 Elnagar 等 Legendre 伪谱来源（DOI `10.1109/9.467672`），可用于解释离散方法家族，但不能证明当前零阶控制、离散动力学或特定 `N` 的收敛阶。

本项目目前在同一 `t_f=78.35 s` 上观察到 `N=30/40/60` 燃耗继续变化，而 `N=20` 无可接受解。这只是探索性证据，说明 `N=30` 尚不足以形成网格无关结论。它不证明单调收敛，也不能区分离散化误差、求解容差、模型松弛误差和真实可行域边界。

## 3. 推荐的 Stage B1 数学分层

### 3.1 两个模型、两个身份

1. `legacy_tf81_v1`：完全保留原始手写 `N=30, t_f=81 s` 模型和 `400.7 kg` 数值回归。其已知干重违反应作为审计结果保留，不能通过改名或容差隐藏。
2. `physical_free_tf_v1`：独立参数化研究模型，显式施加 `m_k >= m_dry`，并将 `N` 与 `t_f` 作为实验输入。其结果不得回写成手写基线的新版本。

对给定 `(N,t_f)`，若现有变换和线性化条件保持，内层仍可作为固定终端时间凸子问题求解。因为 `dt=t_f/N` 同时进入动力学、质量界线性化和目标，直接把 `t_f` 与状态/控制一起作为决策变量通常引入乘积；在没有额外变换或逐次凸化证明前，不应称整体问题为 SOCP。

### 3.2 外层一维搜索的正确定位

外层搜索是构造参考解的稳健第一步：

1. 先在版本化区间 `[t_min,t_max]` 上做粗网格扫描，保留每个尝试，不删除失败点。
2. 仅以通过物理审计的最优状态作为目标函数样本；求解器异常、数值不确定和经认证不可行必须分开编码。
3. 在候选盆地内做有界细化，但不能未经证据假设燃耗关于 `t_f` 全局单峰。至少绘制完整扫描并对局部极小点和状态跳变做检查。
4. 边界点使用更严格容差和第二求解器复核。`SolverError` 只表示该次算法未返回可接受证书，不等于物理不可行。
5. 报告时间区间、粗细步长、停止条件、求解器版本、容差和全部尝试数，使最优 `t_f` 可重算。

对论文而言，外层搜索结果应表述为“在给定搜索域、离散网格和审计规则下的最佳已验证离散解”，直到获得更强的全局性或连续时间证据。

## 4. 每个网格必须独立优化终端时间

固定 `t_f` 比较不同 `N` 会把时间选择误差和网格误差混在一起。推荐序列为 `N in {20,30,40,60,80,...}`，并对每个 `N` 独立执行同一协议的终端时间搜索，得到 `(t_f^*(N), J^*(N))`。不得先在 `N=30` 找到 `t_f`，再把该时间强制用于所有网格并宣称最优值收敛。

每个候选解至少记录和审计：

- 求解器原始状态、迭代数、原始/对偶残差和对偶间隙；
- 燃耗、终端质量、全节点最小干重裕量；
- 终端位置/速度误差和全部 SOC 最大违反；
- 离散动力学缺陷；
- 在更密的独立检查网格上重构后的连续区间缺陷与路径约束违反；
- ECOS 与 Clarabel 对同一标准化锥问题的状态、目标和关键残差差异。

建议预先冻结收敛判据，而不是看到结果后选阈值。例如对连续三个已审计网格，要求 `J^*` 相对变化、`t_f^*` 变化、状态/控制轨迹范数差和密集检查缺陷同时低于预注册阈值。若控制存在切换或尖峰，仅比较节点值会漏检，应比较保持/插值后的区间行为。

Richardson 外推或经验收敛阶只有在离散方案阶次已知、网格加密比例一致且解处于渐近区间时才可使用。若目标非单调、活动集变化或状态在可行/失败间跳变，应报告“未观察到网格收敛”，而不是给出虚假的外推精度。

## 5. 状态与失败分类

Stage B1 至少区分以下互斥结果：

| 分类 | 含义 | 可否用于目标比较 |
|---|---|---:|
| `verified_feasible` | 求解器返回可接受状态，且独立物理与数值审计全部通过 | 是 |
| `solver_infeasible_certificate` | 求解器返回可接受的不可行证书；仍应记录另一求解器是否一致 | 否 |
| `solver_indeterminate` | inaccurate、迭代上限或无法形成可靠证书 | 否 |
| `solver_error` | 异常、崩溃、接口错误或无结果 | 否 |
| `physical_violation` | 数值状态看似成功，但干重、终端、锥或密集区间审计失败 | 否 |

只有 `verified_feasible` 能进入燃耗最小值。不可行证书与算法失败必须分别计数；将二者合并会人为改变可行时间域。目标超出初始可用推进剂几毫克也不能仅凭显示精度通过，应由预注册的质量容差和高精度复核决定。

## 6. 与顶级期刊水平的差距

### 6.1 近期必须补齐

- **全文证据差距**：自由时间和 hp/mesh 专项来源目前仅核验元数据。正式写作前要获取全文，逐页提取问题假设、终端时间策略、离散方法、收敛声明和实验规模。
- **连续时间差距**：节点可行不等于节点间可行。需要密集重采样或高精度传播，报告动力学缺陷和路径约束峰值。
- **网格差距**：当前探索结果明显依赖 `N`。需要每网格独立优化 `t_f`、至少三次连续加密满足预注册门槛，或诚实报告未收敛。
- **证书差距**：ECOS/Clarabel 目标一致不是物理可行性证明。必须同时比较规范化模型、求解状态和独立残差。
- **模型差距**：显式干重约束修复了已知物理违反，但还需审计推力线性化/松弛紧性及所有连续区间约束。

### 6.2 可能形成贡献但尚未成立

- 利用 `t_f` 仅改变数值而不改变稀疏拓扑，复用装配映射、排序或符号分析；贡献需要分阶段计时和数值等价证据。
- 用已验证网格序列构造终端时间 continuation/warm-start；必须与冷启动、同等容差和完整失败率比较。
- 从外层搜索升级到连续终端时间 SCP；必须说明信赖域、虚控制/罚函数、停止条件和收敛失败案例，并以外层参考解交叉验证。
- 构造带后验缺陷估计的自适应网格；必须证明细化指标确实降低连续时间误差，而非只降低节点目标。

外层搜索、换用 Clarabel 或增加 `N` 单独看都不是足够的新颖性。更可信的论文叙事是：原始手写固定时间基线提供独立工程锚点；物理修正模型揭示旧数值基准的边界；结构复用与经审计的自由时间/网格流程共同提供可复现、可部署的改进。

## 7. 推荐实验矩阵与声明边界

| 因素 | 最低设计 |
|---|---|
| 模型身份 | `legacy_tf81_v1` 与 `physical_free_tf_v1` 明确分列 |
| 网格 | 至少 4 个 `N`，并继续加密直到收敛或资源上限 |
| 终端时间 | 每个 `N` 使用相同预注册搜索域和停止规则独立优化 |
| 求解器 | ECOS 与 Clarabel；状态和审计结果均保留 |
| 消融 | 固定 81 s、仅加干重、加干重+自由时间、再加结构优化 |
| 证据 | 所有时间尝试的逐条记录、派生 summary、digest 和 claim ledger |

在 Stage B1 完成前允许的表述是：探索扫描发现 `t_f=81 s` 在显式干重模型下未获可接受解，约 `78 s` 附近存在已审计候选解，并观察到显著网格依赖。禁止表述为“已求得连续时间全局燃料最优解”“`399.733 kg` 是最终最优值”或“较手写基线提升某百分比”，因为模型物理可行域和离散精度均已改变。
