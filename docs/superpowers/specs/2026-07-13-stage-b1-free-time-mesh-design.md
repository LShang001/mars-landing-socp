# Stage B1 物理修正自由终端时间与网格收敛设计

## 1. 研究问题与资产边界

Stage B1 回答一个比旧标称值更严格的问题：加入显式干重约束后，在给定离散网格上，哪些终端时间可由独立求解器得到并通过物理审计的轨迹；这些网格最优值是否随网格加密趋于稳定。

`MarsLanding/MarsLanding.c` 保持为 `legacy_tf81_v1`：N=30、tf=81 s、作者原始独立人工 CRS→CCS 构造。Stage B1 不修改、生成、导出或替换该文件。新模型命名为 `physical_free_tf_v1`，是并列的参数化研究模型。两者的数值不可在没有模型标签的情况下混称为同一基准。

## 2. 数学边界

对固定 `(N, tf)`，内层问题保持 SOCP。状态定义在 `k=0,...,N`，控制只定义在 `k=0,...,N-1`，避免为不存在的终端控制区间支付代价。步长为 `dt=tf/N`，目标为 `min sum(sigma_k * dt)`；燃料仍由 `m0-exp(z_N)` 独立重算。

内层包含：离散位置、速度和对数质量动力学；初末状态；下滑角锥；推力锥；现有推力/质量线性化包络；质量上界；以及所有节点上的显式 `z_k >= log(m_dry)`。加入干重约束不等于证明原线性化松弛精确，最终轨迹仍必须经过独立残差审计。

`tf` 进入动力学系数、质量包络和参考轨迹，因此整个自由时间问题不宣称为单个 SOCP。Stage B1 采用版本化、确定性的外层搜索，每个候选 `tf` 调用一个固定时间 SOCP。

## 3. 软件边界

- `experiments/physical_model.py`：不可变模型配置、固定时间 CVXPY SOCP 构造、ECOS/Clarabel 适配器和参数化物理审计。
- `experiments/free_time_search.py`：候选网格、状态分类、逐网格搜索和跨求解器核验；不得把异常改写为不可行。
- `experiments/studies/free_time_mesh_v1.json`：模型版本、网格、时间范围、细化步长、求解器和容差的唯一 manifest。
- `experiments/run_free_time_mesh.py`：运行入口，逐次追加 JSONL，失败也写记录。
- `experiments/aggregate_free_time_mesh.py`：从 JSONL 重算每个网格的最佳已审计点、求解器差异和网格差异。
- `tests/test_physical_model.py`、`tests/test_free_time_search.py`：TDD 合同和数值小规模/标称回归。

旧 `MarsLanding/mars_params.py` 继续描述 legacy 参数，不改造成可变全局对象。新配置显式复制公共物理常数，并带 `model_id`，防止运行时修改 legacy 全局变量。

## 4. 外层搜索与状态语义

初版使用 manifest 明确定义的嵌套等距网格，不假设燃料关于 `tf` 光滑或单峰：先扫描闭区间粗网格，再只围绕当前最佳已审计点扫描更细网格。所有候选时间经十进制定点量化，候选去重并排序，保证跨平台样本标识稳定。

每个 N 独立完成全部搜索，禁止用 N=30 的最佳时间直接评价其他网格。候选结果分类为：

- `success`：求解器返回允许状态，且独立审计全部通过；
- `physical_violation`：存在解向量，但任一物理/残差门槛失败；
- `solver_infeasible`：求解器明确返回 infeasible；
- `solver_inaccurate`：仅返回 inaccurate 状态，不进入确认性最优集合；
- `solver_error`：异常、无解向量或未知状态。

`solver_error`、`solver_inaccurate` 和 `solver_infeasible` 永不互相替代。搜索区间边界若只有错误而无明确不可行证据，只能报告“未解析边界”。

## 5. 物理与离散审计

审计使用结果自身的 `N`、`tf` 和配置，不调用 legacy 的全局 `N/dt`。至少重算：终端位置/速度、三组动力学缺陷、两个 SOC 违反量、干重/初始质量违反量、推力线性化包络违反量，以及目标积分与终端质量燃料差异。

网格收敛还需做独立的密集区间缺陷检查：将每段常值控制下的解析离散状态用于段内采样，检查质量和下滑角；不能仅检查节点。若现有离散动力学不能给出可信段内模型，报告该限制，不把节点可行称为连续时间可行。

## 6. 双求解器与收敛声明

ECOS 用于完整搜索；Clarabel 至少复算每个 N 的最佳点、相邻时间点及边界代表点。确认点必须同时满足：

1. 两个求解器均为 `success`；
2. 燃料、终端质量和审计残差差异不超过 manifest 容差；
3. 原始记录完整保留迭代数、状态、版本和错误类型。

首版确认性网格固定为 N=20、30、40、60；增加 N=80 必须创建新的 manifest 版本，不能改写首版。只有连续三个加密网格的最佳燃料和最佳 `tf` 变化均落入预先登记阈值，才可写“观察到网格收敛”。否则报告网格敏感性，不外推连续时间最优值。

## 7. 证据冻结与论文门槛

逐候选 JSONL 是权威记录；summary、表和图都是可重算派生物。冻结时记录 manifest/raw/gzip/summary SHA-256，并为每个论文数字增加 claim ledger 条目。图必须同时展示：各 N 独立的燃料-时间可行点、失败分类、每网格最佳点和跨求解器差异；不得只展示最漂亮的一条曲线。

在上述证据未冻结前，`tf=78.35 s, N=30, fuel≈399.733 kg` 仅为探索性观察，不得写成最终最优或顶刊贡献。论文必须并列说明 legacy `400.7 kg` 数值一致但违反干重约束，以及新模型改变了物理与离散定义。

## 8. Stage B1 完成条件

- 新模型与 legacy 资产边界有自动测试，`MarsLanding.c` 内容及构建来源保护继续通过。
- 外层搜索、状态分类、参数化审计和双求解器适配器有 RED→GREEN 证据。
- 每个预注册网格独立寻优，原始失败未丢弃，summary 可从 raw 重算。
- ECOS/Clarabel 确认点通过物理审计，或明确记录无法确认的原因。
- 网格结论使用预注册阈值，不把探索值包装为确认性结论。
- 论文、图、claim ledger、项目工作流文档和 PDF 同步生成并完成逐页视觉核查。
