\section{结论}

本文以火星动力下降段为工程背景，围绕序列锥规划（SCP）框架下的二阶锥规划（SOCP）轨迹优化问题，完成了从数学建模、求解器验证、数值精度分析到嵌入式部署的全链路研究。以下从核心成果、工程价值、数值精度发现和未来工作四个方面进行总结。

\subsection{核心成果}

在建模层面，本文在 SCP 框架下建立火星动力下降轨迹优化问题的 SOCP 数值形式，涵盖动力学离散化、推力与方向约束、燃耗目标和松弛变量。桌面 C 原型由应用代码手写稀疏矩阵，其中等式约束 Jacobian 矩阵包含 733 个非零元，不等式约束 Jacobian 矩阵包含 403 个非零元。该应用路径在运行时不使用 CVXPY 或 YALMIP 建模层，但仍链接 ECOS 和系统数学库。

在数值对比方面，本文保留两组证据边界：三个 C ECOS 目标可由仓库构建产物直接运行；CVXPY+ECOS、CVXPY+Clarabel 和 CasADi+IPOPT 则来自 Python comparison 静态记录。两组在一位小数燃料目标值上均记录为 400.7 kg，可作为实现回归参照。该一致性不表示高精度逐位相等，也不能证明 SOCP 模型物理正确、求解器普遍可靠或凸化无损；当前标称解的干重下界违反仍需修正并重新审计。

在部署方面，当前成果是桌面 Linux 上的 C+ECOS 原型。冻结报告记录 Intel N150 项目主机上 Release AVX 目标在 setup 后 1000 次求解的 CPU 时间均值为 7.361 ms；没有逐次 raw samples，不支持 p99 或 WCET。目标飞控验证仍是后续工作。

\subsection{工程价值}

本项目的全部源代码已开源发布于 GitHub 仓库 \texttt{LShang001/mars-landing-socp}，旨在为 GNC（制导、导航与控制）工程师提供一个可阅读、可验证、可修改的轨迹优化参考实现。与学术界常见的基于 MATLAB 或 Python 建模工具的原型代码不同，本项目的 C 语言实现不依赖任何商业求解器或闭源库，从矩阵装配到求解算法全部透明可审查，特别适合航天嵌入式软件开发人员的学习和工程参考。

在文档建设方面，项目提供了完整的 \texttt{AGENTS.md} 开发指南、三篇经验总结文档以及十九项详细的工程陷阱记录，覆盖了从问题建模、稀疏矩阵构造、求解器调试到精度分析的全过程。这些文档不仅记录技术方案，更注重呈现决策过程和失败教训，力求使读者能够复现整个开发路径。

\subsection{数值精度发现}

在数值精度方面，仓库中的 Clarabel f32 放宽容差配置与 f64 默认配置分别记录 1880+ kg 和 400.7 kg，名义差异约 375\%。由于精度和容差同时变化，该观测不能单独归因于浮点精度，也不能直接形成硬件选型结论。

质量采用 $z=\ln(m)$ 表示时，局部关系 $dm/dz=m$ 意味着在质量约 1500 kg 附近，指定的 $z$ 误差换算为质量误差时乘以约 1500。该结果是局部单位换算，不是自动微分、梯度或 Hessian 灵敏度的实验证据。

当前构建中，标量与 AVX/FMA 目标的平均记录相差约 2.4\%，标量目标略快。缺少硬件计数器和受控频率实验时，不能把这一差异归因于内存、缓存或指令调度；具体瓶颈及优化方向仍待性能计数验证。

\subsection{局限与未来工作}

本文的研究存在若干可拓展的方向。首先，当前模型未考虑大气阻力影响。尽管火星大气密度仅为地球的约 1\%，但高空高速条件下大气扰动对轨迹的累积影响仍可能不可忽略，后续可在动力学模型中增加阻力项并进行敏感性分析。

其次，本文采用质点动力学假设，忽略了飞行器姿态与质心运动的耦合效应。引入六自由度动力学模型，将姿态控制力矩与推力矢量偏转约束纳入优化框架，是提升模型保真度的自然延伸。

第三，本文固定终端时间进行求解，将时间网格作为先验输入。自由终端时间优化问题具有更强的实际意义，允许着陆器在燃耗最优的前提下自主决定任务时长，同时也对离散时间网格的缩放与自适应策略提出了新的挑战。

第四，当前模型未包含障碍规避约束。实际着陆场景中可能需要避开地形障碍或满足特定的视线角约束，这类非凸约束可以通过线性化或凸松弛的方式嵌入 SCP 迭代框架。

第五，本文的所有计算和分析均基于桌面环境，尚未进行硬件在环验证。将生成的 C 代码移植到真实的飞行控制器或实时仿真平台进行闭环测试，是验证算法实际可用性的必经步骤。

火星动力下降与其他推进着陆问题可能共享部分数学结构，但本文没有对再入制导、运载火箭回收或其他载具完成模型与实验验证。跨场景适用性属于后续研究，需要为新动力学和约束重新建立证据。

综上，本文提供火星动力下降 SOCP 的桌面 C+ECOS 原型、分组数值记录和证据审计基础设施。物理可行标称解、同配置精度实验、硬件瓶颈归因、目标飞控移植和跨场景验证仍是开放工作。
