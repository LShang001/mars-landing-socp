# 火星着陆一致性修复设计

## 目标

消除项目源代码、项目手册和论文之间已验证的模型元数据漂移，同时保留
`MarsLanding.c` 中人工编写的 CRS 到 CCS 矩阵构造作为独立的原始基准实现。

## 边界

- 手写矩阵版继续独立维护其物理参数与矩阵装配逻辑；不从 CasADi 输出生成或
  替换该实现。
- CasADi 自动版只用于独立验证手写版，不拥有手写版的实现权威。
- `mars_params.py` 是 Python/自动生成链路的参数来源，不再声称是全部 C/Python
  实现的唯一来源。

## 修复内容

1. 新增轻量 Python 一致性检查，静态读取手写 C 的关键参数和维度宏，并与
   `mars_params.py`、自动生成头文件的公共模型常量比较。
2. 将该检查接入 `ci/validate.sh`，使参数或维度漂移在求解器运行前失败。
3. 更正 README 与 AGENTS 中的约束维度、重力符号、自动生成器行序以及参数来源
   说明。
4. 更正论文中与现有实现矛盾的矩阵维度和本机基准单位；性能表保留其已记录的
   毫秒数据，叙述不再称为微秒或 ARM Cortex-A72 实测。

## 验收标准

- 一致性检查通过，且故意修改关键常量时会失败。
- 手写、自动、AVX 与标量 ECOS 目标仍构建并给出 400.7 kg。
- 文档的等式/不等式维度分别统一为 223/341，自动生成器行序与代码一致。
- 项目不再将手写矩阵描述为由自动生成链路派生。
